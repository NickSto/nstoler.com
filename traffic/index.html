Traffic Monitor Project

Because existing free traffic monitoring services are limited and because I wanted to learn how they work, I wrote my own. My system logs visits to my webpages and displays them in an easily-browsable format.

The system is composed of:

  • A resource embedded on every page which, when requested, logs visit data.

    Technology: Perl CGI scripting and JavaScript.

  • A database the visit data is logged to, and which can be queried to display the data.

    Technology: MySQL

  • An internal browser to format and display visit data for easy visualization.

    Technology: Perl CGI scripting, HTML5, and CSS

The primary limitation was that my shared hosting service does not provide sufficient access to the webserver to allow me to write a system to log traffic on the per-request level. So instead I had to rely on the requested pages themselves performing the logging via the embedded resource.

Result

The final product is a scalable system, in the sense that no special consideration needs to be made to enable logging on each page. To add the page to the system, one must only link to the external resource and include the standard JavaScript file with site-wide custom scripts.

The browser interface is also important, showing the data in an easily-comprehended layout and allowing navigation back through the entire visitor history.
